# esr_neigh

## Description

`esr_neigh` is the APB-accessible ET System Register block for a CORE-ET
neighborhood. It is a clean SystemVerilog translation of the original
`rtl/shire/esr/esr_neigh.v` plus the generated `esr_neigh_auto.v` body.

The block stores neighborhood configuration registers, exposes live read-only
status inputs, and implements the custom ICache error logging and debug
`resethalt` update behavior used by `neigh_channel`.

## Parameters

| Parameter | Default | Description |
|---|---:|---|
| `AdWidth` | `esr_pkg::EsrNeighApbPpAddrWidth` (15) | APB address width including the two privilege-prefix bits. The original neighborhood instance uses `ESR_NEIGH_APB_AD_PP_WIDTH=15`. |
| `SetsPerSubBank` | `shirecache_pkg::SetsPerSubBank` | Preserved original parameter; not consumed by this ESR block. |
| `NumReqqEntries` | `shirecache_pkg::ReqqDepth` | Preserved original parameter; not consumed by this ESR block. |

## Ports

### System and DFT

- `clk_i`: neighborhood LV clock.
- `rst_c_ni`, `rst_w_ni`, `rst_d_ni`: active-low cold, warm, and debug reset domains. They preserve the original separate `reset_c`, `reset_w`, and `reset_d` domains.
- `test_en_i`: preserved original test-enable port. The translated write-preview primitive has no direct test-enable input; this has no functional effect in normal operation.
- `dft_i`: consolidated DFT reset-bypass struct. `dft_i.scanmode`/`scan_reset_n` replace original `dft__reset_byp`/`dft__reset` for the generated latch-reset repeaters.

### ESR outputs and status inputs

The block outputs stored configuration for minion boot address, MPROT,
VMSPAGESIZE, IPI redirect PC, HACTRL, PMU control, neighborhood chicken bits,
ICache error-log control, texture control, and texture image-table pointer.

Read-only inputs are sampled during APB read setup for HASTATUS0,
AND/OR-tree L0, texture status, and live HASTATUS1 busy bits. HASTATUS1 error
and exception bits are sticky: minion input bits set them, and APB writes can
clear them when the corresponding minion input is low.

### APB interface

The APB slave uses the original two-phase APB timing:

1. setup phase: `apb_psel_i=1`, `apb_penable_i=0` decodes address and captures read data or write enables;
2. access phase: `apb_psel_i=1`, `apb_penable_i=1` exposes `apb_prdata_o`/`apb_pslverr_o` and commits writes.

`apb_pready_o` is tied high. Invalid addresses produce `apb_pslverr_o=1` in the access phase.

## Primitive mapping

The original autogenerated ESR latch-write path used `LATCH_P2`,
`et_clk_gate_n`, `esr_clk_gate`, resettable latches, and `rst_repeat`. The
translation uses:

- `prim_write_preview_en` for the APB write-data preview path;
- `prim_write_commit_rst_en` for latch-backed ESR register commits;
- `prim_rst_sync` for the original DFT-aware latch-reset repeaters.

No RAM or SRAM DFT primitive is used because the original `esr_neigh` block has
no RAM instances.

## Differences from original CORE-ET

| Difference | Rationale |
|---|---|
| Active-high `reset_c/reset_w/reset_d` are active-low `rst_c_ni/rst_w_ni/rst_d_ni`. | Project reset convention; reset-domain separation is preserved. |
| Individual `dft__reset_byp`/`dft__reset` are consolidated into `dft_pkg::dft_t`. | Project DFT convention. `scanmode=1` with `scan_reset_n=0` matches original DFT reset assertion. |
| Autogenerated Verilog macros/includes are translated into explicit SystemVerilog and package constants. | Project style and Verilator/Yosys compatibility. |
| Latch/write macros are represented by named write-preview/write-commit primitives. | Required translation policy for generated CSR/ESR latch-write paths. |
| Type and address constants live in `esr_pkg`; ICache error structs are reused from `icache_pkg`. | Avoids duplicate incompatible typedefs while preserving original packed layouts. |

No intentional functional divergence from the original `esr_neigh` behavior is
introduced.
