# esr_spio

Service-processor ESR block for standalone Minion debug status/control.

## Description

`esr_spio` is the small APB-accessible ESR bank used by the original
`standalone_minion` shell to expose service-processor debug control and
status:

- `spdmctrl` — debug mode control outputs toward the service processor
- `sphastatus` — sticky/live status driven from Minion debug state

This is a direct translation of the original CORE-ET `esr_spio` block,
including its split reset domains:

- `rst_c_ni` resets the APB decode/enable staging
- `rst_d_ni` resets the ESR storage registers
- `rst_w_ni` is preserved on the interface for shell parity, but unused here

## Parameters

- `AdWidth`: APB address width including PP bits. Default `4`.
- `SetsPerSubBank`: preserved original parameter, unused by this block.
- `NumReqqEntries`: preserved original parameter, unused by this block.

## Ports

- `clk_i`
- `rst_c_ni`, `rst_w_ni`, `rst_d_ni`
- `test_en_i`
- `svcproc_dbg_status_i`
- `spio_rc_resumeack_i`
- `esr_spdmctrl_o`
- `esr_sphastatus_o`
- scalar APB slave interface:
  - `apb_paddr_i`
  - `apb_pwrite_i`
  - `apb_psel_i`
  - `apb_penable_i`
  - `apb_pwdata_i`
  - `apb_pready_o`
  - `apb_prdata_o`
  - `apb_pslverr_o`

## Integration Notes

- The APB address map matches the original SPIO ESR block:
  - `0xA` — `spdmctrl`
  - `0xB` — `sphastatus`
- `sphastatus.error` and `sphastatus.exception` are sticky-on-set from
  `svcproc_dbg_status_i` and are cleared by APB writes with the corresponding
  status bits written low, matching the original shell behavior.
- `spdmctrl.clrresethaltreq`, `setresethaltreq`, `ackhavereset`, and
  `resumereq` are one-cycle pulse fields, matching the original ESR macros.

## Differences From Original

- Reset polarity follows project convention (`*_ni`, active-low async assert)
  instead of the original active-high reset ports.
- The original autogenerated include/macros are translated into explicit
  SystemVerilog `always_ff` / `always_comb` logic.
