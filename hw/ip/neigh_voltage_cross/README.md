# Neighborhood Voltage Crossings

This IP translates the CORE-ET neighborhood high-voltage/low-voltage crossing wrappers:

- `neigh_hi_voltage_cross`: high-voltage/shire-side push to low-voltage/neighborhood-side pop.
- `neigh_lo_voltage_cross`: low-voltage/neighborhood-side push to high-voltage/shire-side pop.

The wrappers are protocol adapters around the original neighborhood semi-synchronous voltage crossings. Packet and pulse paths use the `prim_fifo_semisync_hiv` / `prim_fifo_semisync_lov` technology primitives, which are standalone-cosim proven replacements for original `vcfifo_wr_hiv_ss` / `vcfifo_wr_lov_ss`. Level/status paths preserve the original `semisync_reg_wr_hiv` / `semisync_reg_wr_lov` behavior inline in the wrapper. Existing `neigh_hv_logic_*` support leaves are reused for original push/edge helper logic.

## Parameters

| Parameter | Module(s) | Default | Meaning |
|---|---|---:|---|
| `FifoDepth` | both | `3` | Translates original `NUM_ELEMS`; all audited neighborhood instances use `SHIRE_HI_VOLTAGE_FIFO_SIZE = 3` or `SHIRE_LO_VOLTAGE_FIFO_SIZE = 3`. |
| `CrossClockVoltage` | `neigh_lo_voltage_cross` | `1` | Parameter form of original `NEIGH_NO_LO_CROSS_CLOCK_VOLTAGE`. `1` enables LV-to-HV FIFO crossings; `0` directly wires the bypassed payload/status paths. |
| `StubMinions` | `neigh_lo_voltage_cross` | `'0` | Parameter form of original `SIM_ALLOW_STUB` / `STUB_MINIONS` path. When all bits are one, the TBOX acknowledge crossing output is forced to zero. |

Shared dimensions and local translated types live in `neigh_voltage_cross_pkg`:

- `apb_to_neigh_t` mirrors original `APB_to_neigh_t` field order.
- `bpam_run_control_neigh_t` mirrors original `bpam_run_control_neigh_t` field order.
- `minion_chicken_bits_vc_t` mirrors the six-bit original voltage-crossing chicken-bits payload.

## Common interface conventions

Both modules use project naming conventions:

| Translated port | Original port(s) | Meaning |
|---|---|---|
| `clk_push_i` / `clk_pop_i` | `clock_push` / `clock_pop` | Semi-synchronous 1:1 DLL-controlled crossing clocks. |
| `rst_push_ni` / `rst_pop_ni` | `reset_push` / `reset_pop` | Ordinary data/status reset domains, active-low in the translation. |
| `rst_c_push_ni` / `rst_c_pop_ni` | `reset_c_push` / `reset_c_pop` | APB/BPAM control reset domains, kept separate from ordinary reset. |
| `dft_hv_i` / `dft_lv_i` | `dft__reset_byp_hv/lv`, `dft__reset_hv/lv` | Consolidated DFT reset bypass controls. `scanmode` maps to `dft__reset_byp_*`; `scan_reset_n` maps to inverted original `dft__reset_*`. |

All ready/valid packet paths keep the original semantics: data is accepted on the push side when `valid/push` and the returned ready are high; the pop side exposes show-ahead data with `valid` and consumes when the parent ready/pop input is high. Several original one-shot pulse paths tie FIFO `pop` to FIFO `valid`; the translation preserves that self-consuming behavior.

## `neigh_hi_voltage_cross` ports and paths

Additional reset domains: none beyond the common `rst_*` and `rst_c_*` domains.

| Path | Direction | Protocol / behavior |
|---|---|---|
| Static ESRs: `shire_id`, `neigh_id`, `tbox_id`, `esr_minion_features`, `chicken_bits`, `esr_shire_coop_mode` | HV to LV | Level-style semisync register, sampled every cycle with `ct_one_hv`. |
| Configuration ESRs: `tbox_en`, `neigh_t0_en`, `neigh_t1_en` | HV to LV | Level-style semisync register. |
| Interrupt levels: `int_mtip`, `plic_meip`, `plic_seip`, `esr_to_neigh_ipi_msip` | HV to LV | Level-style semisync registers. |
| `icache_f0_sram_resp_*` | HV to LV | `prim_fifo_semisync_hiv` ready/valid payload FIFO. |
| `neigh_sc_rsp_*` | HV to LV | `prim_fifo_semisync_hiv` ready/valid response FIFO. |
| `flb_l2_neigh_resp_*` | HV to LV | Self-consuming pulse FIFO; `valid_sync` is the pop-side FIFO valid. |
| `apb_esr_req` | HV to LV | Push generated by `neigh_hv_logic_apb_req`; pop side holds the last request and clears `psel`/`penable` after an accepted beat. |
| `esr_to_neigh_ipi_trigger` | HV to LV | Pulse FIFO; output is zero except on the pop-valid cycle. |
| `uc_to_neigh_fcc` + target | HV to LV | Push generated by `neigh_hv_logic_uc_fcc`; output is zero except on the pop-valid cycle. |
| `dmctrl` + `bpam_run_control` | HV to LV | Push generated by `neigh_hv_logic_bpam`; BPAM payload commits on valid, and `dmctrl_sync_o.write_enable` is the valid pulse delayed one cycle, matching original `dmctrl_we`. |
| `esr_icache_prefetch_start` + config | HV to LV | Self-consuming pulse FIFO; start output is the pop-side valid pulse. |
| Status-monitor debug static inputs | HV to LV | Level-style semisync register. |

## `neigh_lo_voltage_cross` ports and paths

Additional reset domains:

| Port | Original | Meaning |
|---|---|---|
| `debug_rst_push_ni` / `debug_rst_pop_ni` | `debug_reset_push` / `debug_reset_pop` | Reset domains for TBOX ack, status monitor, and debug tree crossings. |
| `rst_w_icache_ni` / `rst_w_icache_sync_no` | `reset_w_icache` / `reset_w_icache_sync` | Warm I-cache reset repeater. The translated output is active-low (`*_no`), so cosim compares it against inverted original active-high output. |

| Path | Direction | Protocol / behavior |
|---|---|---|
| `icache_f2_sram_req_*` | LV to HV | `prim_fifo_semisync_lov` ready/valid payload FIFO. |
| `apb_esr_rsp` | LV to HV | Push on `apb_pready`; pop side uses `neigh_hv_logic_apb_rsp` to zero the response when no valid beat is present. |
| `bpam_rc_tbox_ack` | LV to HV | Change-detect FIFO in the debug reset domain, or zero when `StubMinions` is all ones. |
| `flb_neigh_l2_req_*` | LV to HV | Self-consuming pulse FIFO. |
| `neigh_sm_signals` | LV to HV | FIFO pushes whenever ready, matching the original ready-to-push feedback. |
| `esr_and_or_tree_l0` | LV to HV | Change-detect FIFO in the debug reset domain. The remembered value resets to all ones to force a post-reset update. |
| `esr_icache_prefetch_done` | LV to HV | Change-detect FIFO with original reset defaults (`s1=1`, `ff=0`) to force a post-reset push. |
| `esr_icache_err_detected` / `esr_icache_err_logged` | LV to HV | Semisync pulse register followed by `neigh_hv_logic_icache_err` edge-holding helper. |
| `rst_w_icache_*` | LV to HV | `prim_rst_sync` reset repeater using the HV DFT domain. |

When `CrossClockVoltage=0`, the original `NEIGH_NO_LO_CROSS_CLOCK_VOLTAGE` bypass behavior is used for payload/status paths. The warm I-cache reset repeater remains active because it is outside the original compile-time bypass.

## Integration notes and constraints

- The semi-synchronous FIFO contract assumes the original neighborhood 1:1 push/pop clock relationship with DLL-controlled phase offset. Do not replace these sites with `prim_fifo_async_*` in shared RTL.
- Preserve reset-domain separation when instantiating these wrappers from `neigh_channel`: `rst_*`, `rst_c_*`, `debug_rst_*`, and `rst_w_icache_*` have different original ownership.
- `dft_hv_i` must drive original HV-side reset bypass semantics and `dft_lv_i` must drive original LV-side reset bypass semantics. The FIFO primitive maps the push/pop DFT domain according to HIV vs LOV direction.
- The wrappers do not instantiate RAMs and do not add SRAM DFT clock override ports.

## Differences from the original CORE-ET modules

| Difference | Rationale |
|---|---|
| Active-high resets are exposed as active-low `_ni` / `_no` ports. | Repository reset convention; original reset domains remain separate. |
| Four original DFT reset pins are consolidated into `dft_hv_i` and `dft_lv_i`. | Repository DFT convention; cosim maps `scanmode=dft__reset_byp` and `scan_reset_n=~dft__reset`. |
| `_ss` FIFOs are technology primitives (`prim_fifo_semisync_hiv/lov`). | Keeps phase-controlled semi-sync behavior behind a technology seam and avoids substituting GCD async FIFOs. |
| Original `NEIGH_NO_LO_CROSS_CLOCK_VOLTAGE`, `SIM_ALLOW_STUB`, and `STUB_MINIONS` compile switches are explicit parameters. | Repository rule: functional feature switches become parameters, not synthesizable `ifdef`s. |
| Local type names live in `neigh_voltage_cross_pkg` instead of `soc.vh` includes. | Repository package/type convention; packed field order is preserved and covered by cosim. |

## Verification

- Unit DV: `make -C hw/ip/neigh_voltage_cross/dv clean test` — 41 checks covering both wrappers, reset/DFT paths, FIFO backpressure, APB, I-cache, BPAM, IPI/FCC, prefetch, FLB, and error/status paths.
- RTL cosim: `neigh_hi_voltage_cross` — 144,496 aggregate output-bit comparisons, zero mismatches.
- RTL cosim: `neigh_lo_voltage_cross` — 59,744 aggregate output-bit comparisons, zero mismatches.

Both cosims drive every practical input port low and high/non-zero through directed phases plus randomized 1:1-clock traffic. All outputs are packed into comparison vectors and compared every cycle.
